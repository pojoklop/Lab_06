## Задача 1 — Обработка массивов (2.5 балла)

### Условие

Дан двумерный массив. В первой строке массива указаны номера столбцов.  
Вводится значение `K` — номер столбца матрицы.  
Упорядочить строки матрицы в соответствии с порядком значений в `K`-м столбце (по возрастанию).

---

### Оптимизация

| Приём | Суть | Эффект |
|---|---|---|
| Пузырёк с флагом `swapped` | Если за проход не было обменов — выход досрочно | Лучший случай O(n) вместо O(n²) |
| Индексная сортировка | Меняем местами только `int`-индексы, не строки | Меньше операций копирования памяти |
| Поиск колонки K один раз | Перед сортировкой, не внутри цикла сравнений | Убирает O(n²) лишних поисков |

---

### Наборы тестов

#### Тест 1 — базовый

**Ввод:**
```
Строк: 4
Столбцов: 3

 1  2  3
 9  5  1
 3  8  4
 6  2  7
K = 2
```

**Вывод:**
```
Матрица после сортировки по столбцу 2:
1    2    3
6    2    7
9    5    1
3    8    4
```
*Объяснение:* сортируем по столбцу №2 (позиция колонки j=1): значения 2, 5, 8 → строки идут в порядке 2 < 5 < 8.

---

#### Тест 2 — уже отсортированные данные (проверка ранней остановки)

**Ввод:**
```
Строк: 4
Столбцов: 2

 1  2
 3  5
 7  9
10 15
K = 1
```

**Вывод:**
```
Матрица после сортировки по столбцу 1:
1    2
3    5
7    9
10   15
```
*Примечание:* флаг `swapped` остался `false` после первого прохода — алгоритм завершился за 1 итерацию.

---

#### Тест 3 — K не найден в заголовке

**Ввод:**
```
Строк: 3, Столбцов: 2
1  2
5  3
K = 9
```

**Вывод:**
```
Ошибка: столбец с номером 9 не найден в заголовочной строке.
```

## Задача 2 — Обработка строк (2.5 балла)

### Условие

В необработанном тексте имеется информация о количестве магазинов **«Пятёрочка»** в каждом городе.  
В тексте имеются и числовые данные, связанные со **средним чеком** покупателей.  
Нужно написать программу, которая выведёт на экран **сколько магазинов** в указанных городах.  
Склонение города не менять. Регулярные выражения **не использовать**.

---

### Оптимизация

| Приём | Суть | Эффект |
|---|---|---|
| Один проход для токенизации | Текст разбивается на слова за O(n) по длине | Нет вложенных `find()` в цикле |
| Один линейный проход по токенам | Пары «город — число» ищутся за O(m), где m — кол-во токенов | Вместо многократных `substr`/`find` по всему тексту |
| Признак «число = чек» по следующему токену | Смотрим вперёд на 1 токен, не назад | O(1) на токен, без дополнительных проходов |

---

### Наборы тестов

#### Тест 1 — пример из задания

**Ввод:**
```
В славном городе Челябинске 385, при этом средний чек составляет 503 рубля. Средний чек 527 р. в Ростове-на-Дону 405 магазинов.

```
*(пустая строка — завершение ввода)*

**Вывод:**
```
Количество магазинов «Пятёрочка» по городам:
Челябинске: 385
Ростове-на-Дону: 405
```

---

#### Тест 2 — несколько городов

**Ввод:**
```
В Москве 250 магазинов, средний чек 850 рублей. В Казани 47, средний чек 620 р.

```

**Вывод:**
```
Количество магазинов «Пятёрочка» по городам:
Москве: 250
Казани: 47
```

---

#### Тест 3 — текст без магазинов

**Ввод:**
```
Средний чек покупателя составляет 500 рублей.

```

**Вывод:**
```
Информация о магазинах не найдена.
```
